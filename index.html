For my project I decided to examine trends in the place of origin (fabric) vs the location of where ceramics have been excavated in the Classical world (so the Mediterranean region, primarily). While taking the Etruscans and Early Rome course we talked a lot about how Greek workshops were making pots in Athens but specifically in Etruscan styles to be exported, and I became interested in examining what made people favor Greek-produced Etruscan-styled pottery over, say, Etruscan-produced Etruscan style pottery-- and then more generally, what makes the people in a site favor foriegn pottery, which is more expensive and harder to obtain?

The primary dataset is the Beazley Archive Pottery Database (BAPD), collected and maintained through the Oxford University Classical Art Research Center. The BAPD contains data on ceramic fabric (origin), location found, style of pot, and estimated dating of the ceramic, among others. [https://www.carc.ox.ac.uk/XDB/ASP/searchOpen.asp]

I was unsatisfied with the BAPD location data so I correlated it with data from the Pleiades location data. (This is still a work in progress and I'm considering shifting to another location dataset because the Pleiades one also had lots of names-- but I will still have to use Pleiades for the BAPD entries without a name.) I specifically used the location dataset as opposed to the place dataset because only the location dataset contains data for real world sites and has latitude and longitude recorded. Places can have a location and locations can have a place but its not one-to-one; Atlantis is an ancient place without a recorded location (because it doesn't exist) and there are archaeological sites at locations that haven't been correlated with a known ancient place. [https://pleiades.stoa.org/] [https://github.com/isawnyu/pleiades.datasets/tree/main/data/csv]\

I hoped to use a dataset of ancient ports but it was not publically available and I haven't been granted access yet. :(

In the process of cleaning the data and generating some early graphs I realised that I needed to also be considering the dating of an object, since sociopolitical events in the classical world will of course impact the rate of export/import for pottery. For example, during the Peloponessian War Athenian potters will be less concerned with sending pots to Italy...

Moving forward, for the final, I aim to:

  • Rework how I structure the location data
  • Be able to generate maps (geopandas time...)
  • Bring in other pot datasets (like from [https://archaeologydataservice.ac.uk/archives/view/hayton_eh_2007/downloads_data.cfm] and [https://arachne.dainst.org/], will also probably have to scrape data from museum websites...
  • Factor in time data

And generate models that:

  • Predict the ratio of local pottery vs imported pottery based off location data, time data, and population(?) data | regression model
  • Predict the origin of imported pottery | classification model

Reformatting¶

Since all the dates for these artifacts are estimates, I will separate the Date column into two columns-- the upper estimate for when the piece was made and the lower estimate for when the piece was made.

There are two anomalies in the date formatting, and both deal with only one date being given. In both cases I simply duplicate the date that is given-- this means the earliest date and the latest date will be identical for now.

Then I just split the Date string into the earlier and later dates and add to their respective columns. I delete the original date column when I'm done.

for i, date in bapd["Date"].items(): # loop over rows in Date Column
    if not pd.isnull(date): 
        if ( "to" not in str(date) ) or ( len(date.split(" to ")) ) == 1: # two anomalies in format: either listed as a single  or date == " to -300" 
            date = re.sub('[a-zA-Z]', '', str(date)).strip()
            bapd.at[i,"Date"] = f"{date} to {date}" 
        dates = bapd.at[i,"Date"].split(" to ")
        bapd.at[i,'Earliest Date'] = dates[0]
        bapd.at[i,'Latest Date'] = dates[1]

columns = ["Country", "Region", "Site", "Site, specific"]
    
for i, location in bapd["Provenance"].items(): # loop through the entire provenance column with location data

    if not pd.isnull(location):
        location = re.sub('[^a-zA-Z\d\s,]', '', location).strip() # keep only alphabetic characters and commas
        location = location.split(",") # split into descending sites
        
        for j, column in enumerate(columns): # go through the new columns in order and add each location in order
            if j + 1 > len(location): # makes sure the "add to columns" loop only runs while index present in both columns and location lists
                break
            elif j <= 3: # for the first three columns only add one piece of data
                bapd.at[i, column] = location[j].strip()
            else: # if the location gets very specific just place all the higher specifications into the "specific" column. I didn't want to have 5-7 columns for the few cases with 7 locations listed
                bapd.at[i, column] = location[j:]

nearest port¶

Calculate the distance between two latitude and longitude points:

In [49]:
# https://stackoverflow.com/questions/55227963/calculate-distance-from-latitude-longitude-data-from-dataframe-pandas

def haversine_np(lon1, lat1, lon2, lat2):
    lon1, lat1, lon2, lat2 = map(np.radians, [lon1, lat1, lon2, lat2])

    dlon = lon2 - lon1
    dlat = lat2 - lat1

    a = np.sin(dlat / 2.0) ** 2 + np.cos(lat1) * np.cos(lat2) * np.sin(dlon / 2.0) ** 2

    c = 2 * np.arcsin(np.sqrt(a))
    km = 6367 * c
    return km
